# Evidence Thresholds

## Purpose

Evidence thresholds define what level of proof is required before a finding can influence the Operator Score or appear in a client report.

## Evidence classes

### Class A — Direct evidence

Directly observable and verifiable.

Examples:

- Page content
- Visible CTA
- Missing service page
- Review count
- Page layout
- Form behavior
- Public Google Business Profile fields
- Public social activity

Use for high-confidence findings.

### Class B — Comparative evidence

Observable relative to competitors or benchmarks.

Examples:

- Competitors have stronger review volume
- Competitors have more service-specific pages
- Competitors show stronger portfolio proof
- Competitors rank more visibly for target searches

Use for medium-to-high confidence depending on sample size.

### Class C — Pattern evidence

Based on common patterns and visible indicators.

Examples:

- Generic messaging likely reduces differentiation
- Weak CTA likely reduces action rate
- Lack of project proof likely reduces trust

Use for medium confidence.

### Class D — Inferred evidence

Plausible but not directly verified.

Examples:

- Missed calls may be causing lost leads
- No CRM may create follow-up loss
- No analytics may prevent optimization

Use for low-to-medium confidence and mark as validation required.

### Class E — Client-provided evidence

Information supplied by the client.

Examples:

- Monthly lead volume
- Average job value
- Close rate
- CRM screenshots
- Call logs
- Analytics exports
- Sales pipeline data

Confidence depends on completeness and reliability.

## Minimum report threshold

A finding may appear in the executive report only if it has:

- Class A, B, C, D, or E evidence
- Business impact statement
- Confidence level
- Recommendation

A finding may affect weighted scoring only if it has:

- Class A, B, or E evidence; or
- Class C evidence with clear rationale

Class D findings should usually be treated as validation opportunities, not score penalties.
