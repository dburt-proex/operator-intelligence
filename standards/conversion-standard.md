# Conversion Evaluation Standard

## Purpose

Evaluate whether interested buyers can easily become qualified inquiries.

## Core principle

A business does not need more traffic if it cannot reliably convert the attention it already receives.

## Evaluation domains

### 1. Call-to-action clarity

Signals:

- Primary action is obvious.
- Button language is specific.
- Main action appears near the top of the page.
- Action prompts repeat at logical decision points.
- Action language matches buyer intent.

Strong examples:

- Request a Free Estimate
- Schedule a Consultation
- Request a Service Quote
- Upload Project Photos
- Speak With the Team

### 2. Inquiry capture

Signals:

- Contact form is easy to find.
- Form works on mobile.
- Form captures useful details.
- Form avoids unnecessary friction.
- Confirmation message sets clear expectations.

### 3. Phone path

Signals:

- Phone number is visible.
- Tap-to-call works on mobile.
- Business hours are clear.
- After-hours expectations are clear.
- Follow-up process is defined.

### 4. Estimate workflow

Signals:

- Buyer knows what happens after submitting.
- Buyer can describe the project.
- Buyer can select service type.
- Buyer can indicate urgency.
- Buyer can choose preferred contact method.

### 5. Follow-up system

Signals:

- Confirmation message exists.
- Inquiry is captured in a CRM or tracking sheet.
- Follow-up sequence exists.
- Estimate status can be tracked.
- Reporting exists for submitted inquiries.

## Common findings

- Action prompts exist but are passive.
- Contact form is generic.
- Phone number is visible but not optimized for mobile action.
- No response expectation is stated.
- No structured project intake exists.
- Follow-up process appears manual or untracked.

## Recommended interventions

- Replace passive action prompts with estimate-focused prompts.
- Add service-specific inquiry forms.
- Add mobile-friendly call prompts.
- Add project-detail fields.
- Add CRM capture.
- Add follow-up automation.
- Add conversion tracking.
