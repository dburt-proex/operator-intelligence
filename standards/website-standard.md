# Website Evaluation Standard

## Purpose

Evaluate whether the website supports trust, clarity, conversion, and local business growth.

## Evaluation domains

### 1. First impression

Signals:

- Clear business identity
- Clear service category
- Clear service area
- Strong visual hierarchy
- Professional design
- Immediate next action

Scoring guidance:

- 0: confusing or broken first impression
- 50: understandable but generic
- 100: clear, credible, differentiated, action-oriented

### 2. Navigation

Signals:

- Services easy to find
- Contact page visible
- Phone number visible
- Menu works on mobile
- No unnecessary complexity

### 3. Service architecture

Signals:

- Dedicated pages for core services
- Dedicated pages for high-value services
- Clear descriptions
- Local relevance
- FAQs or objections addressed

### 4. Trust proof

Signals:

- Reviews
- Testimonials
- Project photos
- Before/after proof
- Team photos
- Licenses and insurance
- Guarantees or warranties
- Years in business

### 5. Conversion paths

Signals:

- Primary CTA above the fold
- Phone CTA
- Estimate request form
- Mobile click-to-call
- Sticky CTA if appropriate
- Contact form asks useful qualifying questions
- Response expectation stated

### 6. Mobile experience

Signals:

- Readable text
- Fast perceived loading
- Thumb-friendly buttons
- Click-to-call visible
- No layout breakage
- Forms usable

### 7. Content quality

Signals:

- Specific copy
- Service-area relevance
- Customer pain points addressed
- Differentiation
- Avoids generic filler
- Explains process
- Handles objections

## Common findings

- Hero section is generic.
- CTA is too passive.
- Service pages are too thin.
- Website lacks proof assets.
- Contact path is present but not optimized.
- Mobile experience is functional but not persuasive.
- Copy explains services but does not sell outcomes.

## Recommended interventions

- Rewrite homepage around customer outcome.
- Add service-specific landing pages.
- Add project gallery and before/after proof.
- Add stronger CTAs.
- Add estimate form with qualification logic.
- Add trust section above the fold.
- Improve mobile CTA visibility.
