# Automation Maturity Standard

## Purpose

Evaluate whether repeatable business operations are supported by reliable systems instead of manual memory.

## Automation domains

### 1. Lead capture

Signals:

- Website forms route to a tracked destination
- Google inquiries are tracked
- Facebook inquiries are tracked
- Phone inquiries are tracked or logged
- Source attribution exists

### 2. Lead response

Signals:

- Confirmation message
- Automated email or SMS response
- Internal notification
- Assigned owner
- Response-time standard

### 3. Estimate workflow

Signals:

- Project details captured
- Status tracked
- Follow-up reminders
- Estimate sent status
- Accepted/lost status
- Re-engagement prompts

### 4. Review system

Signals:

- Review request timing defined
- Automated review request
- Review tracking
- Negative feedback routing
- Testimonial capture

### 5. Reporting

Signals:

- Lead count
- Source performance
- Estimate conversion
- Follow-up completion
- Review velocity
- Monthly dashboard

## Maturity levels

| Level | Description |
|---:|---|
| 0 | No automation. Work is handled manually and inconsistently. |
| 1 | Basic notifications exist but no structured workflow. |
| 2 | Some forms or tools capture information, but workflows are fragmented. |
| 3 | CRM or tracking system exists with basic automation. |
| 4 | Lead, estimate, review, and reporting workflows are mostly systematized. |
| 5 | Business has a managed growth operating system with monitoring and optimization. |

## Recommended interventions

- Add CRM or structured tracking database.
- Add form-to-CRM routing.
- Add automated inquiry confirmation.
- Add follow-up reminders.
- Add review request sequence.
- Add monthly reporting dashboard.
- Add owner-level KPI view.
