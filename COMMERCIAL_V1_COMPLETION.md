## 6. Release invariants

The release is valid only while these remain true:

- unknown is never zero
- blocked is uncertainty plus a control state, not automatic failure
- confidence never modifies maturity or priority
- one condition has one weighted owner
- package eligibility precedes assignment
- exactly one primary package exists only for eligible work
- Phase 0 is validation and access, not implementation
- phases 1–5 preserve dependencies and authority
- QC, publication, roadmap approval, proposal acceptance, and implementation authorization are separate
- completion and realized-value evidence are separate
- AI cannot bypass workflow, data, privacy, review, escalation, logging, QA, and failure controls
- unsupported financial, ranking, lead, conversion, market, competitor, savings, ROI, or timeline claims are prohibited
- material changes supersede approved history
- real-client work requires fresh per-engagement evidence and decisions
