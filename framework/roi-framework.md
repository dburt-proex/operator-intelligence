# ROI Framework

Version: v0.3 foundation

The ROI Framework defines how Operator Intelligence models potential business value without overstating certainty.

## Purpose

Clients need to understand why implementation matters.

However, revenue claims must be governed. Operator Intelligence does not invent exact ROI. It models scenarios using stated assumptions, client-provided data, or clearly labeled estimates.

## Core rule

Never present modeled revenue as guaranteed revenue.

Use this language:

- Scenario
- Estimate
- Modeled potential
- Assumption-based projection
- Directional upside

Avoid this language unless supported by verified client data:

- Guaranteed return
- Will generate
- Proven revenue increase
- Exact financial impact

## ROI model inputs

| Input | Source priority |
|---|---|
| Monthly website visitors | Analytics preferred; otherwise unknown. |
| Monthly leads | CRM or owner estimate. |
| Current close rate | CRM preferred; otherwise owner estimate. |
| Average job value | Client-provided. |
| Lead source | CRM, forms, call tracking, or unknown. |
| Response time | CRM/call logs preferred; otherwise owner estimate. |
| Estimate follow-up rate | CRM preferred; otherwise unknown. |
| Review conversion rate | Review request tool or manual tracker. |

## Basic lead-value formula

```text
Estimated Monthly Revenue from Leads = Monthly Leads × Close Rate × Average Job Value
```

Example:

```text
40 leads × 30% close rate × $2,500 average job = $30,000 modeled monthly revenue
```

## Incremental opportunity formula

```text
Modeled Incremental Revenue = Additional Leads × Close Rate × Average Job Value
```

Example:

```text
5 additional leads × 30% close rate × $2,500 average job = $3,750 modeled monthly revenue
```

## Follow-up recovery model

Use when leads or estimates are not consistently followed up.

```text
Recovered Opportunity = Open Estimates × Recovery Rate × Average Job Value
```

Recovery rate must be labeled as an assumption unless client data exists.

## Review impact model

Reviews are not modeled directly as revenue unless supported by client data.

Instead, describe expected effects:

- Increased trust
- Improved local competitiveness
- Stronger Google Business Profile performance
- Better conversion support
- More proof for website and sales process

## Scenario levels

Use three scenarios when enough input data exists.

| Scenario | Use |
|---|---|
| Conservative | Minimal improvement assumptions. |
| Moderate | Reasonable improvement assumptions. |
| Aggressive | Strong improvement assumptions with higher uncertainty. |

## Confidence labels

| Confidence | Use when |
|---|---|
| High | Inputs come from analytics, CRM, or verified reports. |
| Medium | Inputs come from owner estimates and visible evidence. |
| Low | Inputs are assumed for scenario planning only. |

## ROI note for reports

Use this disclaimer in client-facing reports:

> Financial projections in this assessment are scenario models, not guaranteed outcomes. Actual results depend on market demand, implementation quality, response time, pricing, close rate, seasonality, operational capacity, and client execution.
